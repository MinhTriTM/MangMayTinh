# Q5: Viết lại @Mục Tiêu.txt

## Câu hỏi của người dùng
viết lại @Mục Tiêu.txt , môn này thi trắc nghiệm ABCD, tôi muốn chia làm 2 phần 1 là file chứa câu trắc nghiệm mỗi chương có từ 250 đến 500 câu trắc nghiệm, kèm theo đó là 1 file lý thuyết tổng quan, tổng hợp của chương đó, ghi đầy đủ kiến thức của chương, bao gồm kiến thức, mở rộng chi tiết nhất, khoảng 10k hay 40k character.

## Suy nghĩ của Agent
- Người dùng muốn thay đổi mục tiêu dự án từ môn Lịch sử Đảng sang môn Mạng Máy Tính.
- Cấu trúc yêu cầu mới: Mỗi chương chia thành 2 file riêng biệt:
    - File 1: Lý thuyết tổng hợp (10k-40k ký tự), chi tiết, mở rộng.
    - File 2: Ngân hàng câu hỏi trắc nghiệm (250-500 câu/chương).
- Cần cập nhật vai trò chuyên gia thành chuyên gia Mạng Máy Tính.
- Cần cập nhật quy tắc đặt tên file để phân biệt giữa file lý thuyết và file trắc nghiệm.
- Đã tiến hành soạn thảo lại nội dung file `@Mục Tiêu.txt` để phản ánh đúng các yêu cầu mới về số lượng câu hỏi, dung lượng lý thuyết và định dạng trắc nghiệm ABCD.

## Câu trả lời
Tôi đã cập nhật file `@Mục Tiêu.txt` theo yêu cầu mới của bạn. 

**Các thay đổi chính:**
1. **Môn học:** Chuyển sang Mạng máy tính.
2. **Cấu trúc:** Mỗi chương hiện được tách thành 2 file: một file lý thuyết chuyên sâu (10k-40k ký tự) và một file trắc nghiệm (250-500 câu).
3. **Quy tắc đặt tên:** Sử dụng hậu tố `_ly_thuyet_` và `_trac_nghiem_` để dễ quản lý.
4. **Nội dung:** Yêu cầu tập trung vào kiến thức chi tiết, mở rộng và các câu hỏi trắc nghiệm ABCD có đáp án rõ ràng.

Bạn có thể kiểm tra file `@Mục Tiêu.txt` để bắt đầu triển khai các bước tiếp theo.
